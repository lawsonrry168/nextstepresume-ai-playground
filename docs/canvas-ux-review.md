# Canvas Studio 與預覽區 UX 完整審視

> 兩大部份：(A) Canvas Studio 佈局與工具擺位、(B) 預覽區浮動工具列與 Dark Mode。
> 標記 ✅ = 本輪已實作；⏳ = 建議後續實作（附具體方案）。

---

## A. Canvas Studio

### A1. 版型入口斷層 ✅
**問題**：Studio 的「版型佈局」只有 6 種排版預設（現代版型/經典單欄…），31 款履歷版型完全看不到 — 用戶必須離開 Studio 回到預覽 dock 的「佈局」面板才能換版型，心智模型斷裂。
**已實作**：工具面板「版型佈局」頂部加入 **31 款版型下拉選單**（依 Notebook/Bureau/Studio 三家族分組、顯示雙語名稱、即時切換）。排版預設下拉保留在其下，職責分明：上面選「設計」、下面選「排版」。

### A2. Dark mode 色調 ✅
**問題**：冷藍黑（`#0a0e14` 底 + `#2a3344` 邊框 ×27 處）+ 高對比文字 `#e8edf4` + 高飽和紅 `#e8635a`/金 `#b8943a` — 色溫互相打架、長時間觀看刺眼。
**已實作**：整套換成「Night Desk Warm 暖石墨」色盤（118 處替換）：
- 面：`#171512 → #1b1916 → #211e1a → #232019 → #2a2721`（暖炭五階）
- 邊框：`#38342c`（暖灰、低對比）
- 文字：主 `#e5e0d4`（暖米白，降亮度）、次 `#a29c90`
- Accent：紅 `#d47f72`（柔和 terracotta）、格線藍 `#6e8396`（降飽和）、金 `#b09355`、mint `#a4c6a6`
- 陰影冷藍環 → 暖中性
與日間的復古文具色盤同一 DNA（米白紙的夜間版），全域一致。

### A3. 工具面板資訊架構 ⏳
**現況問題**（截圖審視）：
1. 右側工具欄一次展開全部區段（版型佈局/對齊/視圖/圖層/尺寸/頁面），需要大量滾動；「對齊」九宮格佔位大但低頻使用
2. 圖層面板（左）每個圖層 7 個控制鈕（眼/鎖/上下/前後/頁面指派）常駐 — 視覺噪音大
3. 佈局動作 16 顆按鈕平鋪（單欄/置中/緊湊/雙欄/三欄/側欄/等距/頁高/頁尾/等寬/等高/靠左/頁中/靠右/鏡像/貼格/安全區）無分組層次

**建議方案**：
- 區段記憶摺疊：預設只展開「版型佈局 + 頁面」，其餘記住用戶的開合狀態（`navSectionOrder` 已存在，加 `collapsedSections` 到 CanvasStudioUiState）
- 圖層列控制鈕 hover 才顯示（常駐只留眼睛 + 頁碼），減少 70% 噪音
- 佈局 16 鈕分三組小標題：「整頁排版」（單欄/雙欄/三欄/側欄/置中/緊湊）、「分佈」（等距/等寬/等高/頁高/頁尾/鏡像）、「對齊」（靠左/頁中/靠右/貼格/安全區）
- 選中 section 時右欄自動捲到「尺寸」區（情境跟隨）

### A4. 滾動與伸縮 ⏳
- 畫布縮放已有（54%/適合/±），建議加 `Ctrl+滾輪` 縮放 + `空白鍵拖曳` 平移（Canva 慣例）；快捷鍵浮層已有基礎
- 底部頁條與縮放列在窄視窗會擠壓 — 建議合併為單條自適應 dock

## B. 預覽區（非 Canvas）

### B1. 浮動工具列可讀性 ✅
**問題**：右上浮動 dock 只有 4 個裸圖示（佈局/調色/眼睛/下載）+ 一個無標示數字（54）— 新用戶無法辨識，ATS 分數像謎題。
**已實作**：每個圖示下方加**常駐文字標籤**（佈局/主題/檢視/匯出），ATS 徽章加上「ATS」字樣 + 完整 tooltip；按鈕加大為全寬直欄、加 `aria-pressed`。

### B2. 浮動面板行為 ⏳
**現況**：點圖示彈出 300px 面板，遮住預覽右緣；ATS 分數同時出現在 dock 和面板底部（重複）。
**建議**：
- 面板改為從右緣推入（預覽區自動縮排，不遮內容）— 復用 ResizableSplitPane 機制
- 面板記住上次開啟的區段；Esc 關閉已有 ✅
- 「檢視」面板裡的自由版面/JD 高亮開關與 Studio 內開關狀態同源（已同源）但文案不一致（「自由版面」vs「畫布」）— 統一詞彙表

### B3. 預覽區域本體 ⏳
- 預設縮放應「適合寬度」而非 100%（初載常只看到左上角）；`previewAutoFit` 已存在，建議預設開
- 多頁時預覽缺頁碼指示 — 建議浮動「第 1/2 頁」膠囊（Studio 已有，預覽區沿用）
- 匯出按鈕散落三處（頂欄「匯出」、dock「匯出」、Studio header）— 保留 dock 為唯一入口，其餘收斂為捷徑同一選單
